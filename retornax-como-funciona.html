Como o Retornax
funciona por dentro

A Retornax é uma plataforma de recorrência que transforma clientes ocasionais em clientes recorrentes — via WhatsApp, no automático. Aqui você entende as tecnologias por trás disso tudo.

A analogia

Pense como um restaurante

Cada parte do Retornax tem uma função específica — como os setores de um restaurante que trabalham juntos para servir o cliente.

🗄️
Supabase
O banco de dados + sistema de login
É o "arquivo" do restaurante — guarda tudo: clientes, histórico de mensagens, sessões do WhatsApp e o perfil completo de cada empresa. Também controla o login seguro.
Analogia: É o caderno de reservas + fichário de clientes + cofre com as senhas dos funcionários.
⚙️
Railway
O servidor backend
É a "cozinha" do sistema — fica rodando 24h por dia, processa pagamentos, envia as mensagens e executa as tarefas automáticas. Também roda o Baileys (WhatsApp) de cada cliente diretamente no servidor.
Analogia: É a cozinha do restaurante — o cliente não vê, mas é onde tudo é preparado e entregue.
📱
Baileys
A ponte com o WhatsApp
É o "garçom" que entrega a mensagem. Roda direto no servidor — sem passar por terceiros, sem custo extra por cliente.
Analogia: É o motoboy de entrega — o restaurante prepara o pedido, o motoboy leva até o destino.
📁
GitHub
Onde fica o código
É o "manual de receitas" do sistema — guarda todo o código do produto. Quando atualizamos algo, o Railway busca automaticamente a versão nova.
Analogia: É o livro de receitas — quando o chef melhora uma receita, todos na cozinha usam a versão nova automaticamente.

Perguntas frequentes

Dúvidas que podem surgir

Respondendo as perguntas mais comuns sobre a parte técnica do negócio.

Se o Railway cair, o sistema para? ▼

O Railway tem 99,9% de uptime garantido. Em caso de queda (raro), o sistema fica indisponível por alguns minutos e volta automaticamente. O Supabase e o GitHub Pages continuam funcionando — só o envio de mensagens fica pausado. As sessões do WhatsApp (Baileys) são restauradas automaticamente pois ficam salvas no Supabase.

Os dados dos clientes estão seguros? ▼

Sim. O Supabase usa criptografia em todos os dados e tem backup automático diário. Cada empresa só acessa os próprios dados — há isolamento total. O sistema segue as normas da LGPD (Lei Geral de Proteção de Dados).

Por que Baileys e não Z-API? ▼

Já migramos! O Baileys é open source e roda direto no Railway — sem custo por instância, sem dependência de terceiro. A Z-API cobrava R$99,99 por cliente/mês. Com Baileys, esse custo é zero. A sessão fica salva no Supabase e persiste mesmo após reinício do servidor.

Quanto tempo leva para atualizar o sistema? ▼

Atualizações do site (painel, landing page) ficam no ar em segundos via GitHub Pages. Atualizações do servidor (Railway) levam 2-3 minutos para o deploy automático. Todo o processo é automatizado — basta salvar o código no GitHub.

É possível escalar para 1.000 clientes com essa estrutura? ▼

Com a estrutura atual (Railway + Supabase + Baileys), sim — o banco suporta milhões de registros sem custo adicional e o Railway escala automaticamente. Com Baileys já migramos para zero custo variável por instância. O próximo upgrade seria aumentar o plano do Railway conforme o volume de mensagens crescer.
